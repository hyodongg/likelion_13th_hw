⊙효율적인 코드

■ template 상속

  • template 상속이란?
    • 반복되는 HTML 코드를 한 곳에 놓아두고 다른 템플릿에서 불러와서 재사용하는 방식
    • 모든 HTML마다 head, nav등이 계속 반복되므로 효율 향상을 위해
    • 공통 부분은 base.html에 저장후 나머지 페이지에는 그걸 상속해서 필요한 내용만 추가
  • runserver 시 base.html을 찾을 수 없는 에러
    • mainpage.html은 main의 template안에 있고 base.html은 전체 프로젝트용 templates 폴더 안에 있음
    • 따라서 settings.py에 들어가서 찾을 수 있게 해주어야 함!

■ navbar 분리

  • 분리 이유?
    • navbar,footer,sidebar 같은 모든 페이지에 반복되는 HTML은 계속 복붙하지 말고 한 곳에 저장해서 재사용 하는 것이 좋음
  • 파일 이름에 언더바(_)붙이는 이유?
    • 직접 브라우저에 보여지는 파일이 아니라 include로만 쓰는 조각들이어서 HTML조각파일 이라는 개발자들 간의 약속
  • static 이란?
    • 웹페이지에서 사용되는 정적인 파일을 저장하는 디렉토리(css,image)
    • 이 파일들은 서버에서 계산하거나 바꾸지 않고 브라우저에 그대로 전달되어 화면에 보여짐
    • 장고가 또 main만 인식할 수 있으므로 또 경로 설정 해주기!
